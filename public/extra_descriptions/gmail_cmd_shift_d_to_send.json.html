Cmd+Shift+D (Command+Shift+D, ⌘+⇧+D) is the Mac convention for "Send email" in most mail clients — Apple Mail, Microsoft Outlook, Airmail, Spark, and others all use it. But in Gmail's web interface the same shortcut discards the draft instead of sending it. Gmail's actual send shortcut is Cmd+Return (Command+Enter, ⌘+⏎), and Google does not expose the discard shortcut in Gmail's keyboard-shortcut settings, so it cannot be disabled or rebound from within Gmail itself.

This Karabiner-Elements rule remaps Cmd+Shift+D to Cmd+Return whenever a supported web browser is the frontmost application, so muscle memory from other mail clients sends the Gmail message instead of deleting it. Useful for anyone searching for "Gmail send shortcut Mac", "Cmd Shift D discard Gmail fix", or "Gmail keyboard shortcut send email Mac".

Supported browsers (the remap only fires when one of these is frontmost): Google Chrome (including Chrome PWAs), Safari, Arc, Firefox, Microsoft Edge, and Brave.

Limiting the remap to Gmail only (optional)

Because this rule fires in any supported browser, Cmd+Shift+D is remapped on every website you visit — not just Gmail. If you rely on Cmd+Shift+D for something else in your browser (for example, bookmarking all open tabs in Chrome), you can scope the remap to a standalone Gmail web app instead:

  1. Open mail.google.com in Google Chrome.
  2. Click the ⋮ (three-dot) menu at the top-right of Chrome → Cast, save, and share → Install page as app…, name it Gmail, and click Install. (In older Chrome versions this is under Save and share → Install Gmail….)
  3. Launch the new Gmail app once so macOS registers it, then find its bundle identifier by running this in Terminal: 
    osascript -e 'id of app "Gmail"'
    You should see something like com.google.Chrome.app.fmgjjmmmlfnkbppncabfkddbjimcfncm.
  4. Edit the rule's bundle_identifiers list (in ~/.config/karabiner/assets/complex_modifications/ or directly in ~/.config/karabiner/karabiner.json) and replace all the browser entries with a single anchored entry for your Gmail PWA, for example: 
    "bundle_identifiers": ["^com\\.google\\.Chrome\\.app\\.fmgjjmmmlfnkbppncabfkddbjimcfncm$"]

With that change, Cmd+Shift+D still behaves normally in regular browser windows and only gets remapped to Cmd+Return inside the Gmail app.
